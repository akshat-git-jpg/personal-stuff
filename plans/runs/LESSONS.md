# Cross-run executor lessons

Append-only, one line per non-obvious lesson learned from a run — recurring
executor mistakes, plan-shape fixes that prevented them, quirks. The
orchestrator reads this during recon (orchestrate Step 2) and writes to it
after verification (Step 4d), so every batch compounds instead of
rediscovering failure modes through fix-up rounds.

Format: `YYYY-MM-DD <executor> — <lesson>`

## Lessons

2026-07-05 antigravity — first block was a PLAN defect, not an executor one: specced yt-dlp subtitle fetching with `--sub-langs "en.*,en"`, which bursts a request per auto-translated variant and self-inflicts HTTP 429. Recon should check `tooling/cli/` for an existing fetcher (pp-yt-transcript) before speccing new fetch code.
2026-07-05 loop — watch-run.sh/runlog-status.sh used to grep the whole log, so a round-1 BLOCKED false-alarmed every later round; both are now active-segment-aware (after last `ROUND N START`) and the ORCHESTRATOR must append the round marker at dispatch. Regression fixtures: `scripts/fixtures/round2-*.md`.
2026-07-05 antigravity — behaved well under fix-up: honored STOP condition literally, kept scope exactly to plan file lists, resumed cleanly from a revised plan when told to re-read it as source of truth.
2026-07-05 antigravity — ignores run-log discipline on long GUI runs (no heartbeats, DONE lines late or never) even when the prompt demands them; watcher exit 3/4 is unreliable for it. Verify liveness via file mtimes + git log, size the staleness window generously, and treat commits as the real completion signal.
2026-07-05 antigravity — leaves debug dumps in the working tree (session traces, prompt copies, scratch SQL); sweep untracked files at verification before declaring the tree clean.

2026-07-06 antigravity — watch-run.sh exited 4 ("never started") while the run was demonstrably alive (PLAN 033 START logged, commits landing); an mtime+git-log watcher over the repo is the reliable wrapper, with run-log lines only as positive signals, never as absence-evidence.
2026-07-06 antigravity — CORRECTED root cause (owner screenshot next morning): the mid-036 stall was NOT quota death — Antigravity was waiting on a GUI permission dialog ("Allow write access to this path?") because the plan's smoke test wrote OUTSIDE the workspace (~/kb-scratch/). All quota meters were at 100%. Two standing fixes: (1) plans executed by Antigravity must keep every write inside the repo, or the dispatch prompt must warn the owner a permission dialog will appear; (2) before declaring a stalled Antigravity dead, inspect its window (accessibility UI text) — a permission modal produces zero file activity and is indistinguishable from death by mtime signals alone. Also still true: it left an uncommitted partial edit plus a premature edit for a LATER plan (037's CLAUDE.md row) — inspect uncommitted diffs for cross-plan contamination before continuing.
2026-07-06 sonnet — resume prompts must say "branch from the current branch" not a literal branch name: a stale literal name from the batch template confused the 038 handoff (executor correctly deviated, but flagged it). Sonnet subagents otherwise kept perfect run-log discipline and surfaced honest deviations — good fallback lane.
2026-07-06 loop — CRITICAL: synchronous subagents leave the shared checkout on THEIR branch; the orchestrator's follow-up `git merge <that-branch>` becomes a silent self-merge no-op while the working tree still shows all files — verification passes, the branch pointer lies. Plans 037/038 were "verified and pushed" while absent from the pushed line. Rules: (1) `git branch --show-current` before EVERY merge/commit in a shared checkout; (2) subagent prompts must end with "check out the base branch before finishing"; (3) verify pushes by `git ls-tree <pushed-sha> <key path>`, not by working-tree tests. A concurrent session (other account) cherry-picking in the same checkout amplified this — the wt pool exists to end shared-checkout orchestration; use it for the NEXT batch.
2026-07-06 agy — print mode does NOT bind cwd as workspace: without --add-dir the run executes against a scratch dir, loads no repo rules, and roams the home dir under skip-permissions (it opened a keyring file during testing). Also: default --print-timeout is 5m (kills real plan runs — set 180m), CLAUDE.md is never read (repo ships AGENTS.md symlink), and --output-format json (hidden flag) is the only way to get usage + conversation_id.
2026-07-07 agy — headless print mode does ~3 plans in a single "turn" (num_turns:1) then ends its response and stops, even with more plans queued; the run-log discipline degrades (heartbeats/DONE lines stop) but per-plan COMMITS are the reliable progress signal. CRITICAL: `--model "Claude Opus 4.6 (Thinking)"` was NOT honored — the Antigravity Claude quota was at 0%, so agy silently fell back to Gemini 3.1 Pro for the whole build (the envelope does NOT record which model ran; the Antigravity status-bar meter is the only ground truth). When a Claude model is requested with 0% Claude quota, agy returns status:SUCCESS with response:"" and 0 tokens (~5-7s) instead of erroring — indistinguishable from success in the envelope. Fixes: (1) do NOT trust --model; check the Antigravity meter or a cheap probe to confirm the active model before a real run; (2) treat a 0-token status:SUCCESS envelope as a FAILURE, not done; (3) size multi-plan batches to ~3 plans and verify by COMMITS, not the run-log; finish a small remainder directly if agy returns empty.
2026-07-09 agy — Gemini 3.1 Pro (High) executed a 10-step behavior-preserving refactor (055-heygen-web, 787-line monolith → layered endpoints/operations/workflows/cli) in one turn: 10 per-step commits, kept run-log discipline, honored offline-only + no-live-API STOP conditions, and the code passed the merge gate first try (5/5 tests). Fully-inlined plans (endpoint table + code skeletons in the plan body) remain the agy sweet spot for refactors, not just greenfield.
2026-07-09 orchestrate — `node --test <dir>` (directory arg) FAILS on node 22.14 with "Cannot find module '.../test'" (it treats the path as a script, not a test dir). Working forms: `node --test <file>.test.mjs`, a glob, or `node --test` with NO arg from the package dir (auto-discovery). Never author a plan's test_cmd as `node --test some/dir/` — the executor will faithfully copy the broken command into package.json + docs.
2026-07-09 agy — concurrent-process contamination reached the COMMIT level this run: a live `wrangler dev` session on the shared checkout committed unrelated fixes onto the run's branch mid-run, interleaved with agy's commits. agy's own commits stayed correctly path-scoped. Do NOT rewrite history while the concurrent session still holds the checkout; isolate the merge in a worktree instead.
