**Security backlog (all confirmed by reading the code):**
- **Rate limiting / lockout on every password/PIN gate** — analytics, kushal-tools,
  founders (numeric PIN, small keyspace), lists, hyperframes-render all allow
  unlimited online guessing. `SEC-03`. Effort M (KV sliding-window per IP).
- **hyperframes-render SSRF surface** — pasted HTML renders in headless Chrome from a
  `file://` origin on the VPS; can read local files / hit the metadata endpoint /
  reach LAN services and composite the bytes into a downloadable MP4. `SEC-02`.
  Effort M (network-isolate the render container, block `file:`+private IPs, asset allowlist).
- **hyperframes-render insecure defaults** — `APP_PASSWORD` defaults to `"changeme"`,
  session token is a constant non-expiring HMAC, password compared with `===`. `SEC-04`. Effort S.
- **tracker-app `DEV_AUTH` header bypass** — when `DEV_AUTH="1"`, `X-Dev-Email`/`X-Dev-Roles`
  headers mint any role incl. Admin with no credential; one env var away from a prod
  admin bypass. Fix = compile the branch out of prod builds, not just gate on env. `SEC-06`. Effort S.
- **Constant non-expiring session tokens** — founders-tracker (`ok.<hmac>`) and
  kushal-tools (length-oracle password compare); no server-side revocation. `SEC-05`.
  Effort S. (Plan 004 adds the tests these fixes will land against.)
- **personal-dashboard `SESSION_SECRET` hardcoded fallback** — folded into Plan 001 (fail-closed).
- **kushal-docs unbounded R2 uploads + unvalidated object-id** — `PUT /api/blob|thumb|view/:id`
  streams to R2 with no size/content-type check and no UUID validation on the client-supplied key.
  Single allow-listed user, so lower severity. `SEC-07`. Effort S.
