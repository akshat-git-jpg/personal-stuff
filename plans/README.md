## Findings NOT turned into plans (deferred backlog, not rejected)

These are real audit findings the owner chose not to plan yet. Recorded so
they aren't re-audited from scratch — promote any to a plan when ready.

**Security backlog (all confirmed by reading the code):**
- **Rate limiting / lockout on every password/PIN gate** — analytics, kushal-tools,
  founders (numeric PIN, small keyspace), lists, hyperframes-render all allow
  unlimited online guessing. `SEC-03`. Effort M (KV sliding-window per IP).
- **hyperframes-render SSRF surface** — pasted HTML renders in headless Chrome from a
  `file://` origin on the VPS; can read local files / hit the metadata endpoint /
  reach LAN services and composite the bytes into a downloadable MP4. `SEC-02`.
  Effort M (network-isolate the render container, block `file:`+private IPs, asset allowlist).
- **hyperframes-render insecure defaults** — `APP_PASSWORD` defaults to `"changeme"`,
  session token is a constant non-expiring HMAC, password compared with `===`. `SEC-04`. Effort S.
- **tracker-app `DEV_AUTH` header bypass** — when `DEV_AUTH="1"`, `X-Dev-Email`/`X-Dev-Roles`
  headers mint any role incl. Admin with no credential; one env var away from a prod
  admin bypass. Fix = compile the branch out of prod builds, not just gate on env. `SEC-06`. Effort S.
- **Constant non-expiring session tokens** — founders-tracker (`ok.<hmac>`) and
  kushal-tools (length-oracle password compare); no server-side revocation. `SEC-05`.
  Effort S. (Plan 004 adds the tests these fixes will land against.)
- **personal-dashboard `SESSION_SECRET` hardcoded fallback** — folded into Plan 001 (fail-closed).
- **kushal-docs unbounded R2 uploads + unvalidated object-id** — `PUT /api/blob|thumb|view/:id`
  streams to R2 with no size/content-type check and no UUID validation on the client-supplied key.
  Single allow-listed user, so lower severity. `SEC-07`. Effort S.

**Correctness/hygiene backlog:**
- **Tutorial pipeline: 162 avatar + graphics overlay passes** — deliberately deferred from
  plan 011's review (2026-07-05): implementing them before real HeyGen downloads exist would
  mean guessing the file-naming convention. Unblocks when `lib/heygen.py` `TODO[HNS]` stubs
  are filled and step 135 renders its first clips. `PIPE-01`. Effort S-M.
- **telegram-my-planner digest fails silently** — no top-level try/except and no
  `requests` timeouts; a Google/Telegram error kills the cron with only local mail,
  no alert. `BUG-01`. Effort S (wrap `__main__`, send failure to Telegram, add timeouts).
- **hyperframes-render resource leak** — per-render temp dirs never deleted, in-memory
  `jobs` Map never evicted; slow disk/memory fill on the long-lived container. `BUG-02`. Effort S.
- **Node CLIs have no package.json/engines pin** — fine today (builtins only), non-reproducible
  the moment one adds a dep. `DEP-01`. Effort S.
- **`ty/hyperframes-vs-remotion` is marked superseded but still tracked** (incl. ~20 MP4s).
  Plan 003 untracks its media; wholesale archival of the project deferred.

**Tracker-app backlog (2026-07-05 focused audit; promote when wanted):**
- **Per-stage SLA defaults in the PipelineDef** (`slaDays` on `StageDef`) feeding the
  attention panel (016) thresholds per system + auto-suggested ETAs. `TRK-01`. Effort S-M.

## Findings considered and rejected (do not re-audit)

- **Move Workers to a single home / physically integrate `ty/`** — rejected. Every
  external toucher (VPS crons, skill symlinks, the hardcoded-path list in
  `scripts/README.md`) keys on current paths; moves are high-risk, low-benefit.
  `ty/` stays a self-governing subtree (Plan 002 records this decision).
- **Merge the two `decisions.md` logs into one** — rejected for the same reason;
  `ty/` governs itself. Root map delegates instead (Plan 002).
- **Rewrite git history to purge tracked media (BFG/filter-repo)** — rejected.
  `.git` is only 52MB; the accretion is stopped forward-only by Plan 003. History
  rewrite is disruptive and unjustified at this size.
- **Add CI (GitHub Actions)** — rejected. The repo deploys manually and is
  single-owner; `scripts/check-apps.sh` (Plan 004) is the right-weight gate.
- **Build a reconciler for `skills-lock.json` vs the manifest system** — rejected.
  They're two independent mechanisms (`npx skills` plugins vs repo-owned skills);
  Plan 005 documents the split rather than coupling them.
- **CLAUDE.md quality overhaul** — not needed. The existing folder CLAUDE.md files
  (tracker-app, gym-app, personal-dashboard, hyperframes-render) are dense and
  operational, not prose duplicates. Only the three *missing* ones are a gap (Plan 002).
- **Tracker: admin UI for authoring pipeline definitions** (a "system builder") —
  rejected 2026-07-05. Definitions-as-typed-code is the right tradeoff while the
  founder is the only admin: a new system is one ~40-line `PipelineDef` + deploy,
  validated by TypeScript + `validatePipelines()`. A builder UI would re-implement
  that validation for a once-a-quarter action.
