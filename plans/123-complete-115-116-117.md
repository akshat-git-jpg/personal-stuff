### Step 4: Ungate chrome in the 7 cards

For each of `checklist/icon-pills`, `comparison/credits-math`, `comparison/table-rows`, `process/step-flow`, `pros-cons/pros-cons`, `verdict/persona-match`, `verdict/winners-podium`:

```js
// BEFORE
tl.from('#eyebrow', {...}, Math.max(0, firstAt - 0.15));
tl.from('#title',   {...}, firstAt);

// AFTER — chrome enters immediately, independent of beat timing
const CHROME_IN = 0.15;
tl.from('#eyebrow', {...}, CHROME_IN);
tl.from('#title',   {...}, CHROME_IN + 0.15);
```

Read each file and map to its actual chrome element ids (`#eyebrow`, `#title`, `#heading`, `#head`, header rows) — **do not blind-replace across the seven**, they differ. Row/beat tweens keep `firstAt` and their per-beat offsets untouched.

The `BEAT_LEAD_IN` clamp already keeps `firstAt` near 0.3s, so this is defence in depth rather than a live fix — but `pre_beat_render: "chrome"` is currently a false claim for these 7 cards, and this makes it true.
