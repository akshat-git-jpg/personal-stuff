# Rulebook — 010 write-script

Write the narration script for the video. **VO-first**: what you write here is
what the narrator says — the visuals get built to it later.

## Input
`videos/<slug>/source.md` — a topic, brief, notes, or a source article. (If it
doesn't exist, ask the user for the topic/angle first.)

## Output
`videos/<slug>/script.md` — the spoken script as prose (paragraphs, in delivery order).

## Devsplainers-style structure
1. **Cold-open hook** — a surprising claim/tension in the first ~2 sentences (no "in this video").
2. **The one confusing thing** — name the crux (often a single term) the video will resolve.
3. **What you'll learn** — a quick promise of the payoff (optional, short).
4. **Body** — explain via concrete metaphors; introduce recurring characters/objects; let color-coded concepts recur (old vs new vs problem vs payoff).
5. **The catch / the honest bit** — where the marketing overreaches.
6. **Payoff + outro** — why it matters; a clean closing line.

## Voice
Conversational, tight, second-person. Short sentences. One idea at a time (this
makes the next step — beat segmentation — clean). Prefer a vivid metaphor over
jargon; when a technical term is unavoidable, define it in plain words first.

## Then
Proceed to 020 (segment into beats). Keep sentences self-contained so each can
become its own beat/scene.
