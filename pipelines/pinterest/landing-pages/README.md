# landing-pages

Per-niche static landing pages, one folder each. Every folder is an assets-only Cloudflare Worker that serves its `public/` directory. Running `npx wrangler deploy` inside a folder also creates the DNS record and SSL cert.

- `keto-kitchen/` — live at `keto-kitchen.agrolloo.com`.
- `bridebestie/` — the wedding brand, live at `bridebestie.com`. Pages are generated by `build-pages.mjs` (one shared template, per-product copy) into `public/<slug>/index.html`. Product covers come from `../wedding/products` (sibling in this repo).

These are the funnel destinations for the Pinterest pins; pin data and the generators live alongside under `TY/pinterest/`. (Moved here from the `personal-stuff` repo.)

## Generated output is gitignored

The generator (`build-pages.mjs`) is committed; the built per-product pages it writes into `public/<slug>/` are not (they're ~1 MB each of copied covers, and regenerable). `public/index.html` and the homepage assets stay tracked. After editing copy in `build-pages.mjs`, run `node build-pages.mjs` to rebuild locally, then `npx wrangler deploy`.
