# landing-pages

Per-niche static landing pages, one folder each. Every folder is an assets-only Cloudflare Worker that serves its `public/` directory. Running `npx wrangler deploy` inside a folder also creates the DNS record and SSL cert.

- `keto-kitchen/` — live at `keto-kitchen.agrolloo.com`.
- `bridebestie/` — the wedding brand, live at `bridebestie.com`. Pages are generated by `build-pages.mjs` (one shared template, per-product copy) into `public/<slug>/index.html`. Product covers come from `TY/pinterest/wedding/products`, so a full rebuild needs the TY repo present.

## Generated output is committed

The built pages under `public/` are checked in, not gitignored. The deployed site is exactly what's in `public/`, and committing it keeps the repo deployable on its own without re-running the generator. After editing copy in `build-pages.mjs`, run `node build-pages.mjs` and commit the regenerated folders.
