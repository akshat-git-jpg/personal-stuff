# scripts

Repo-wide orchestration scripts in one place, plus an index of everything *outside* this repo that depends on its layout. If you move or rename a folder, this is the list of things that break.

## Scripts here

- `relink.sh` — recreates the per-account skill symlinks in `~/.claude-work/skills` and `~/.claude-personal/skills` from the manifests in `tooling/claude-skills/manifest/`. Idempotent. Run it after cloning, after moving the repo, or after changing skill membership. Resolves the skill store relative to itself, so a repo rename doesn't break it — but the symlinks it already created point at absolute paths, so you must re-run it after a move.
- `regen-mcp-json.sh` — rewrites the gitignored `.mcp.json` at the repo root with this machine's `tooling/mcp` server paths. Run after cloning or moving the repo.
- `vps-sync.sh` — **VPS only.** `git pull`s the VPS clone and rebuilds `~/.claude/skills` (single root account) from `manifest/personal.txt`. The VPS counterpart to `relink.sh`. A `repo-sync` cron runs it every 15 min so interactive Claude (Remote Control / mobile) on the VPS stays current with what you push. Safe to run by hand for an instant sync.
- `check-apps.sh` — Uniform verification runner. Iterates through all apps and runs typecheck/check, lint, and test scripts. Returns exit code 0 if all pass, 1 if any fails.

## External touchpoints (things that hardcode this repo's path)

These live outside the repo and must be updated by hand if the repo moves or a folder is regrouped:

- `~/.zshrc` — a git-identity rule mapping this repo path to the `akshat-git-jpg` GitHub account, and the `ccu-dash` alias pointing at `tooling/cli/ccusage-dashboard/dashboard.mjs`.
- `~/.claude-work/skills/` and `~/.claude-personal/skills/` — symlinks into `tooling/claude-skills/` (regenerated by `relink.sh`).
- `.mcp.json` (repo root, gitignored) — absolute paths into `tooling/mcp` (regenerated by `regen-mcp-json.sh`).
- The `github-router` skill (`tooling/claude-skills/github-router/SKILL.md`) — its path → account rule keys on this repo's path.
- The separate `vps-crons` repo (`~/codebase/vps-crons`) — its `my-planner/run.sh` and `gmail-digest/run.sh` wrappers reference `apps/telegram-my-planner/tools/daily-digest/` and `apps/telegram-email-assistant/digest.sh` inside the VPS clone of this repo.
- The Hostinger VPS — clones this repo to `/srv/projects/personal-stuff/` and `vps-crons` to `/srv/crons/`. Both pull on cron ticks. Gitignored runtime (`.venv`, `token.json`) does not move with `git pull`, so relocate it on the VPS after any folder regroup.

## After cloning on a new machine

```bash
./scripts/relink.sh          # recreate skill symlinks
./scripts/regen-mcp-json.sh  # write .mcp.json for this machine
```
